# Por que fazer o curso de HTML5?

    Para responder esta pergunta precisamos ver a história do desenvolvimento web.

    O reinado do Flash.
    Por alguns anos nesta história o Flash reinou absoluto, tufo que era inovação na web era atrelado a este software. Era 
    uma verdadeira febre. Companhias de todos os tamanhos e alguns sites se tornaram imortais.

    Mas como todo produto tem seu nascimento sua ascensão e também sua queda o número de aplicações criados  em Flas caíram 
    bastante e aplicações móveis não implacaram.

    E o motivo foi o lançamento do primeiro smartphoe de verdade o que triuxe uma maneira diferente de navegar na internet 
    com mobilidade total o iPhone.

    O iPhone foi o grande algoz do Flas uma vez que seus navegadores não rodavam o plugin in Flas Player, vital para o 
    usuário acessar sites criados no software da Adobe.

    O lançamento deste produto fex com que uma linguagem que ainda estava precoce como estimativa de finalização em 2022 
    tivesse seu uso e crescimento acelerado que foi o HTML5 e começou ali um novo reinado. A dinastia do HTML5 sem 
    anecessidade de plugins mais aberto mais democrático se tornou popular em pouquíssimo tempo e logo foi apontado como o 
    Flash killer.

    Porém este reinado não é exercido por apenas um rei. Ele tem outras majestades o CSS# em conjunto com o JavaScript estão 
    levando a web tudo aquilo que o Flash levava, porém roda nos smartphones, tablets e todos aqueles produtos que hoje são 
    obrigatórios ela lares e empresas. Dominar bem o HTML5, CSS3 e JavaScript é essencial para qualquer desenvolvedor web.

    HTML

    HTML - Hyper Text Markup Language

    Linguagem de marcação de hipertexto utilizada para estruturar contúdo em páginas web.

    O HTML foi criado em 1991 por Tim Berners-Lee (físico britânico) no CERN (European Council for Nuclear Research) na Suíça. Inicialmente, o HTML foi projetado para interligar instituições de pesquisa próximas e compartilhar documentos com facilidade.

    Documentos HTML podem ser interpretados por navegadores. A tecnologia é fruto da junção entre os padrões HyTime e SGML.

    HyTime é um padrão para a representação estruturada de hipermídia e conteúdo baseado em tempo.

    SGML é um padrão de formatação de textos. Não foi desenvolvido para hipertexto, mas tornou-se conveniente para transformar documentos em hiper-objetos e para descrever as ligações.

  - Sintaxe do HTML

   Nos comendos são utilizadas duas marcas ou tags que são:

   <> - Tag de abertura
   </> - Tag de fechamento

   Sendo a base estrutural de comando composto por:

    <comando atributo1="valor1" ...atributoN="valorN">
      Content
    </comando>
    
    Exemplo
    
    <p id="paragrafo">
        Hello World
    </p>
    
    As tags HTML não são case sensitive, portanto escrevendo <HTML>, <Html>, <html> ou <HtMl> será interpretado pelo browser
    porém recomendações de padrão de uso direciona a utilizar <html>.
    
    As tags são usadas para delimitar o início e o fim dos elementos na marcação. Texto bruto , texto bruto escapável e 
    elementos normais têm uma marca inicial para indicar onde começam e uma marca final para indicar onde terminam. 
    As tags iniciais e finais de certos elementos normais podem ser omitidas. 
    Aqueles que não podem ser omitidos não devem ser omitidos. Os elementos vazios possuem apenas uma marca inicial; 
    tags finais não devem ser especificadas para elementos void . elementos devem ter uma tag inicial e uma tag final, 
    ou uma tag inicial marcada como de fechamento automático; nesse caso, elas não devem ter uma tag final.
    
    O conteúdo do elemento deve ser colocado logo após a tag inicial (que pode estar implícita, em certos casos ) 
    e logo antes da tag final (que, novamente, pode estar implícita em certos casos ). Os conteúdos exatos permitidos de 
    cada elemento individual dependem do modelo de conteúdo desse elemento, conforme descrito anteriormente nesta especificação. 
    Os elementos não devem conter conteúdo que seu modelo de conteúdo não permita. Além das restrições impostas aos conteúdos 
    por esses modelos de conteúdo, no entanto, os cinco tipos de elementos têm requisitos sintáticos adicionais.
    Elementos vazios não podem ter nenhum conteúdo (já que não há tag final, nenhum conteúdo pode ser colocado entre a tag inicial e a tag final).
    
    Fonte: https://html.spec.whatwg.org/multipage/syntax.html#the-doctype

<!DOCTYPE html>
 - Indica ao browser que o conteúdo do arquivo é uma página html.
   Em HTML, o doctype é o preâmbulo obrigatório "<!DOCTYPE html>" encontrado na parte superior de todos os documentos. 
   Seu único objetivo é impedir que um navegador mude para o chamado "modo peculiar" ao renderizar um documento; ou seja, 
   o doctype "<!DOCTYPE html>" garante que o navegador faça o melhor esforço para seguir as especificações relevantes, 
   em vez de usar um modo de renderização diferente que seja incompatível com algumas especificações.

    O termo meta tag é, na verdade, formado por 2 palavras que se juntam para dar seu significado. E para entender este significado, é preciso entender as 2 palavras, em separado:

Meta é um prefixo grego que significa algo como "após, "que ultrapassa", "que engloba", "que está além". Este não é um prefixo utilizado somente no desenvolvimento para web; você já deve ter ouvido e lido palavras como "metabolismo" e "metamorfose". Este prefixo meta refere-se à "coisas sobre a própria coisa" ou, em outras palavras, dados sobre dados, informação sobre a informação.

Tag é uma palavra do inglês que significa "etiqueta" ou "rótulo". Tanto nos EUA, no Brasil ou qualquer parte do mundo, etiquetas servem para identificar, nomear e marcar algo ou alguma coisa para que possa ser identificado e/ou corretamente catalogado – dentre outras coisas, são tags que descrevem o documento web a qual pertencem ("informação sobre a informação"); as meta tags são para descrever informações sobre sites e páginas que as contém; informam sobre qual conteúdo está ali e mostra informações extras a respeito deste conteúdo.

As metatags são colocadas como elementos-filho de <head>, no HTML, e sua sintaxe obedece à seguinte convenção:

<meta name="nome-da-meta" c />
Meta tags mais usadas

Existem muitas meta tags disponíveis para se usar, entretanto, algumas são de especial importância e devem ser obrigatoriamente inseridas em cada uma das páginas.

Meta tag "description" Esta meta tag descreve o conteúdo do documento.

<meta name="description" c>

Meta tag "keywords" Nesta meta tag você coloca uma lista de palavras-chave.

<meta c>

Meta tag "robots" Serve para controlar os bots de busca quanto à sua tarefa de indexação. Através dos diferentes valores possíveis para seu conteúdo, é possível passar instruções aos bots de busca para dizer qual o critério de indexação foi escolhido para tal ou qual página de seu site. Os valores mais comuns para esta meta tag são:

    all. Valor padrão, significa "vazio" – o robô de busca não recebe nenhuma informação;
    index. Os robôs de busca podem incluir a página normalmente;
    follow. Robôs podem indexar a página e ainda seguir os links para outras páginas que ela contém;
    noindex. Os links podem ser seguidos, mas a página não é indexada;
    nofollow. A página é indexada, mas os links não são seguidos;
    none. Os robôs podem ignorar a página;
    noarchive A página não é arquivada.

color:#93c 12. Breve Histórico HTML5.

Em 2004 a W3C iniciou o deenvolvimento do XHTML dois pontos eram frustados com os rumos do XHTML 
2.0. Um grupo deprofissinais e corporações iniciou o desenvolvimento do que foi chamado de HTML5. 
Este grupo é chamado de Web HipertextApplication Technology Working Group ou WHATWG. 
  
O grupo inicia as especifica&ccedil;&otilde;es que eles chamam de Web Applications 1.0.

Vamos trabalhar com texto de documento iniciando com os 'heds' que são os cabeçalhos,
bastante comum em editores de texto.
Os heds são sub-estilos padrões e restritos reconhecidos pelo navegador por herdados do HTML4,
no qual através de CSS podem ser etilizados através de classes. 

<h1></h1> Aplicado ao texto de maior destaque.Nos 'heds' exitem 6 tipos de tamanhos entre eles:

  <h1> O maior, mais usada em títulos principalmente em otimização SEO.
  <h2> Mais usada em sub-títulos.
  <h3>
  <h4>
  <h5>
  <h6> O menor.
  <hgroup> 
  Esta tag nasceu com o HTML5 para otimizar a semântica dos documentos web, serve para agrupar, reunir 
  os cabeçalhos 'heds'.

color:#93c 30. Usar as tags 'i' e 'em'.

Especificações indica o uso nas seguintes situações de textos:

<i> - Voz alternativa como transliterado palavras estrangeiras, termos técnicos e texto tipograficamente em itálico.

<b>-  estava em negrito, agora para texto “estilicamente deslocado”, como palavras-chave e texto tipograficamente em negrito

<em> - ênfase, agora para ênfase tônica , ou seja, algo que você pronunciaria de forma diferente.

<strong> - <strong>— era para ênfase mais forte, agora para importância forte , basicamente a mesma coisa (ênfase ou importância mais forte agora é indicada por aninhamento) 

<i> e <b> eram elementos de estilo de fonte HTML4 e ainda são usados ​​na apresentação quando apropriado para seguir as convenções tipográficas. Eles agora têm significado semântico, no entanto, e seu estilo pode ser alterado via CSS, o que significa que eles não são apenas de apresentação — <b>, por exemplo, não precisa ser em negrito. Por isso, é recomendável usar classes para indicar o significado para facilitar a alteração do estilo posteriormente.

o <i> elemento

O elemento i representa um trecho de texto em uma voz ou humor alternativo, ou de outra forma deslocado da prosa normal de uma maneira que indica uma qualidade diferente de texto, como uma designação taxonômica, um termo técnico, uma frase idiomática de outro idioma, um pensamento, ou um nome de navio em textos ocidentais.

Para opinião de terceiros.

Use somente <i> quando nada mais adequado estiver disponível — por exemplo, <em> para texto com ênfase, <strong> para texto com importância semântica, <cite> para títulos em uma citação ou bibliografia, <dfn> para a instância definidora de uma palavra e <var> para variáveis ​​matemáticas. Em vez disso, use CSS para colocar blocos de texto em itálico.

Outras coisas que são tipicamente em itálico incluem palavras estrangeiras transliteradas (usando o atributo lang=""), instruções de palco embutidas em um script, alguma notação musical e ao representar texto manuscrito embutido:

Exemplo:

<p>There is a xertain <i lang="fr">je ne sais quoi</i> in the air.</p>

Para o usuário esta notação não há muita relevância mas para o engine de renderização do navegador e semântica são úteis.
